REST API Working

This project is a Node.js-based REST API that allows users to retrieve Quranic verses from a CSV file. It utilizes the Express.js framework for handling HTTP requests and enables cross-origin requests using CORS. The API reads verse data from a file named quran.csv, processes it, and returns the requested chapter and verse in JSON format. The project has been successfully deployed on Render, making it accessible online.

How It Works?

Server Setup:
The API is built using Express.js and runs on a specified port (3000 by default).
It ensures that quran.csv exists before starting; otherwise, the server exits with an error.

Handling Requests:
The API provides a single endpoint:
GET /quran/:chapter/:verse

The endpoint accepts:
chapter (integer) - The chapter number of the Quran.
verse (integer) - The verse number.

Data Processing:
When a request is made, the server reads quran.csv and searches for the requested chapter and verse.

The CSV file is assumed to have data formatted as:
id, arabic_text, chapter_number, chapter_name, verse_number, verse_text
The script extracts relevant data and formats the output.

If verse=0, it returns the entire chapter.

If the verse is found, the API returns a JSON response:

{
"chapter": "Al-Fatiha",
"text": "In the name of Allah, the Most Gracious, the Most Merciful."
}

If the verse is not found, it returns:

{
"error": "Verse not found"
}

Deployment on Render:
The API is hosted on Render, making it publicly accessible.
Render provides automatic deployments, meaning any updates pushed to the repository can be deployed seamlessly.
This ensures the API is available 24/7 with minimal maintenance.

Running the Server Locally:
The server listens on the specified port and logs a message when it starts:
Quran API is running on port 3000
